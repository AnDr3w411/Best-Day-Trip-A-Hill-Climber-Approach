Day-trip-ai-project

Hill climber algorithm to find the best possible day trip

View on GitHub Download .zip Download .tar.gz

Generate the best possible day trip!

This project is meant to generate an optimized day trip using the googleAPI to get places and a hill climbing algorithm to find the trip that takes you to the best places with the shortest travel time. This is a type of traveling salesman problem with a hill climber algorithm used to solve it. With the googleAPI I was able to get all of the places within a specified radius of the types that I am interested in, i.e. museums. I am able to query googleAPI to find the travel time between 2 places and each place has a rating attribute. With a rating and a time the algorithm just goes through a pregenerated trip making changes where ever it can to improve the "happiness" level of the trip.

I decided to do this project because I never no what I want to do and I saw an article saying scientists found the perfect road trip so I wanted to make something that could potentially be useful to find something to do for the day.

You will need a GoogleAPI key. Instructions on how to get one can be found here https://developers.google.com/maps/documentation/javascript/get-api-key Specifically Google Maps Distance Matrix API and Google Places API Web Service are what are used here.
